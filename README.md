# 🍓 酒ログ 🦋 - 日本酒記録アプリ (PWA)

飲んだ日本酒を写真と感想で美しく記録するための、軽量・高機能なプログレッシブ・ウェブ・アプリ (PWA) です。

## 🌟 主な機能

### 1. 酒ログの新規登録
- **銘柄名**: お酒の名前を入力。
- **写真 (最大5枚)**: 複数枚の写真をアップロード。iPhoneでも滑らかにスワイプできる高機能カルーセルを搭載。
- **評価 (5段階)**: 星1〜5で満足度を記録。星の色が水色から紫へグラデーションで変化。
- **味の感想タグ**: 「甘口」「フルーティー」「すっきり」など、豊富なタグから複数選択可能。
- **飲み方 (温度帯)**: 冷酒、常温、ぬる燗、熱燗から選択。
- **特別タグ**: 
  - 🤍 **お気に入り**: また飲みたいお酒をマーク。
  - 🌸 **概念**: 季節や特別なテーマのお酒を分類。
  - 🐡 **山口のお酒**: 地域特有のお酒を分類。
- **その他メモ**: 感想を自由に記録。
- **関連リンク（飲んだお店など）**: 飲んだお店や購入サイトのリンクを保存。

### 2. 記録一覧とフィルター
- **年別・月別フィルタ**: 過去の記録を月ごとに素早く検索。
- **特別タグフィルタ**: 「お気に入り」だけを表示するなど、絞り込みが可能。
- **年別アコーディオン**: 記録数が増えても見やすいアコーディオン形式のリスト。
- **カード表示**: 一覧では正方形の写真で見やすく、詳細では元の比率で閲覧可能。

### 3. データ管理と引き継ぎ
- **エクスポート**: 記録データをJSONファイルとしてダウンロード（バックアップ）。
- **インポート**: 保存したファイルを読み込んでデータを復元。
- **機種変更対応**: 新旧端末間でのデータ移行ガイドをアプリ内に内蔵。

## ⚠️ 【重要】アップデート時の注意点 (iPhone/PWA)

本アプリはブラウザの `localStorage` にデータを保存しています。iOSの仕様上、**ホーム画面からアイコンを削除すると、それまでの記録データもすべて消去されてしまいます。**

アップデート（最新機能の反映）の際は、以下の手順を必ず守ってください。

### ✅ 正しい更新手順
1. アプリを開き、ヘッダー（画面右上）の「データ管理」ボタンの左に **「✨ 更新あり」** というバッジが出るまで待ちます。
2. バッジをタップすると **「新しいバージョンがあります。更新しますか？」** という確認が出ます。
3. **「OK」** を押すと、アプリが自動で再読み込みされ、データはそのままで最新版に切り替わります。

> [!CAUTION]
> **絶対にホーム画面からアイコンを削除して再インストールしないでください。**
> プログラムの更新は、アプリ内の「✨ 更新あり」ボタンから行うのが最も安全です。

## 開発・リリースのルール（改修時は必ず守る）

### バージョンを毎回上げる

**コードを改修したら、その都度バージョン番号を更新する。** 変更の大きさにかかわらず、リリース単位＝バージョンアップとする。省略すると、利用者の端末で更新の検知や更新内容の表示がずれる。

### 更新履歴は「利用者向けの説明」だけにする

`changelog.json` に書く内容は、**アプリを使う人が読んで意味が通る説明**に限定する。

| 書いてよいこと | 書かないこと（例） |
|----------------|-------------------|
| 画面上で何が変わったか、何が直ったか | ファイル名・関数名・ライブラリ名・「リファクタ」「内部処理」など開発者向けの言い回し |
| 操作のしかたが変わった場合は、その要点 | インフラ・キャッシュ・ストレージの仕組みだけの説明（利用者メリットが伝わらないもの） |
| 不具合修正なら、ユーザーが困っていた現象 | 「〇〇を更新した」だけで中身が伝わらない抽象表現の羅列 |

※ 仕様上どうしても技術用語が必要な場合は、**利用者にとってのメリット**が一文で分かるように添える。

## 開発者向け：バージョンアップ時の作業チェックリスト

GitHub などへ反映する前に、**次のすべてを同じバージョン番号（例：`v1.2.4`）にそろえる。**

1. **`sw.js`**  
   `CACHE_NAME`（例：`sake-log-cache-v1.2.4`）を新バージョンに合わせる。これでブラウザが新しい更新を検知する。

2. **`app.js`**  
   `currentVersion` が **2 か所**ある（更新履歴ボタン用・初回アップデート通知用）。**どちらも**同じ文字列にする。

3. **`index.html`**  
   データ管理まわりに表示しているバージョン表記（例：`v1.2.3` のバッジ）を新バージョンに合わせる。

4. **`changelog.json`**  
   新バージョンのキー（例：`"v1.2.4"`）を追加し、`date` と `changes`（配列）を記入する。内容は上記「利用者向けの説明」ルールに従う。

5. **リポジトリへ push**  
   上記をコミットして反映する。
